Лабораторная работа 11: Получение данных из сети и Paging
Ответы на контрольные вопросы

1. Для чего в проекте нужны Retrofit и конвертер Gson? 
Retrofit создает HTTP-клиент и вызовы по интерфейсу, а 	конвертор json  - преобразование тела ответа в объекты 	котлина ччерез Gson
2.Как по интерфейсу с аннотациями @GET и @Query выполняется реальный HTTPзапрос? 
	Retrofit должен сам сгенерировать реализацию интерфейса 	который мы опишем, а когда мы будем вызывать методы, 	библиотека сама откроет соединение, отправит запрос и 	вернет результат.

3. Почему сетевой вызов стоит выполнять в Dispatchers.IO, а обновление UI - в главном потоке?
	Чтобы приложение не зависало до получения ответа 	сетевого запроса, который может занять время.

Ответы на контрольные вопросы 2
1. Зачем нужна пагинация при работе с большими списками?
	Пагинация нужна для того чтобы не ждать долго ответа от 	приложения когда необходимо загрузить допустим сразу 	несколько тысяч записей, с помощью пагинации мы грузим 	записи небольшими отрывками например по 20 штук, а 	следующие будем подгружать когда пользователь 	доскроллит страницу до конца
 2. Какую роль выполняют PagingSource и Pager? 
PagingSource загружает одну конкретную страницу данных 	по номеру, а Pager управляет этим процессом, создает поток 	данных, следит за скроллом.
3. Как в Compose отобразить список из Paging и обработать загрузку и ошибку?
	Отобразить список из Paging можно получив данные через 	метод collectAsLazyPagingItems и после передать в 	LazyColumn
	Чтобы обработать загрузку и ошибку необходимо 	использовать метод loadState, у него есть 2 состояния:
1 - refresh - первичная загрузка, проверяем через when
2 - append - подзагрузка при скролле, проверяем после 	блока items
